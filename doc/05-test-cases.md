### 问卷系统测试用例集

本项目测试旨在验证从问卷创建、发布到填写、统计的全链路业务逻辑。测试过程中需严格比对后端数据库 MongoDB 的状态变化。

#### 场景一：问卷生命周期基础流程 (创建与题目管理)
验证发布者能否成功构建包含异构题目的问卷原型。
- **创建问卷**：调用 POST /api/v1/surveys 接口，输入标题与截止时间，预期返回 200 并生成 DRAFT 状态的问卷 ID。
- **创建题目版本**：调用 POST /api/v1/questions 创建首个题目版本，预期 `questions` 集合新增版本号为 1 的题目文档。
- **题目版本升级**：调用 POST /api/v1/questions/{question_id}/versions 基于旧版本创建新版本，预期新文档 `version = 2`，且 `previousVersionId` 指向 v1。
- **更新基础配置**：调用 PATCH /api/v1/surveys/{id} 接口修改匿名开关与截止时间，预期 MongoDB 中问卷元数据同步更新。
- **装配问卷快照**：通过 PUT 接口向问卷注入 `questionId + version + orderIndex` 结构，预期服务端自动读取题库版本并生成快照。
- **预期结果**：MongoDB `surveys` 集合中对应文档的 `questions` 数组应包含版本引用和 `snapshot` 副本，且字段校验规则落库正确。

#### 场景二：题目分享与权限控制
验证题目所有者能否将题共享给其他用户，以及共享后的访问权限边界是否正确。
- **建立共享**：用户 A 调用共享接口将题目 `q_xxx` 共享给用户 B，预期 `questions` 集合中该题所有版本都出现相同的共享授权记录。
- **共享后可读**：用户 B 调用题目版本链与题目详情接口，预期可以读取被共享题目的版本信息。
- **共享后可用**：用户 B 在自己的问卷中装配该题版本，预期保存成功并生成问卷快照。
- **未共享拒绝访问**：用户 C 未获共享时，调用题目详情或尝试把该题装配进问卷，预期返回 403 或 404 风格错误。
- **修改权限隔离**：用户 B 虽然能使用共享题，但不能创建该题的新版本，也不能修改共享列表。
- **预期结果**：共享只授予“查看与使用”权限，不转移所有权，不放大到版本创建权限。

#### 场景三：题库管理
验证用户能否把题加入题库、从题库移除，并正确浏览自己的题库内容。
- **加入自己的题到题库**：用户 A 把自己创建的题加入题库，预期 `questions` 集合中该题所有版本都出现 `libraryMembers` 记录。
- **加入共享题到题库**：用户 B 在获得共享后，把用户 A 的题加入自己的题库，预期成功且不影响 A 的题库标记。
- **浏览题库**：用户调用题库浏览接口，预期返回自己题库中的所有逻辑题，覆盖“自己创建的题”和“别人共享给自己的题”。
- **移出题库**：用户将题从题库中移除后，题库浏览结果中不再出现该题，但题目详情、共享关系和问卷装配能力仍保持不变。
- **权限校验**：用户不能把自己无权访问的题加入题库。

#### 场景四：动态跳转逻辑 (核心逻辑验证)
验证系统能否根据用户的不同回答，实时且准确地执行预设的题目跳转路径。
- **设置规则**：在问卷中配置逻辑规则，如“若第 1 题选 A，跳转至第 5 题”。
- **组合匹配**：对于多选题，若分别配置条件 `1` 与 `2`，当填写者实际选择为 `1 2` 时，预期不得命中前两条规则；仅当存在条件 `1 2` 时才可跳转。
- **格式校验**：选择题跳转条件统一使用行号表示法，如 `1 3`；若出现重复行号或越界行号，预期保存失败。
- **快照隔离**：若题库中源题已升级为新版本，已保存问卷的逻辑判断仍应以旧快照选项为准。
- **作答演练**：
    1.  模拟填写者选择第 1 题选项 A，预期前端或后端计算引擎返回的下一题游标指向第 5 题。
    2.  模拟选择选项 B，预期游标按自然顺序指向第 2 题。
- **预期结果**：跳转引擎应能覆盖单选、多选及数字填空（如数值 > 100 时跳转）的三类触发场景。

#### 场景五：业务约束与校验 (安全性校验)
验证系统在多维度输入限制下的稳健性，确保无效数据无法落库。
- **必答性校验**：提交一份遗漏必答题的答卷，预期接口返回 422 错误及明确的提示信息。
- **数值边界校验**：在数字填空题输入超出 [min, max] 范围的值，预期提交失败。
- **整数约束校验**：在要求整数的数字题输入小数，预期提交失败。
- **文本长度校验**：在文本题输入长度短于最小值或长于最大值，预期提交失败。
- **多选数量校验**：在要求选择 2 个的题目中仅选 1 个，预期系统阻断提交。
- **匿名开关校验**：当问卷允许匿名时，填写者可选择匿名或实名；当问卷不允许匿名时，匿名提交应被拒绝。
- **提示语校验**：所有阻断信息必须使用“第N题”而不是内部题目标识。
- **截止时间校验**：问卷超过截止时间后，预期系统阻断提交。
- **预期结果**：所有阻断行为均需在 API 返回值中体现，且 `answers` 集合中不得增加无效文档。

#### 场景六：答卷提交与统计聚合 (数据完整性验证)
验证高并发背景下的答卷持久化能力及后期统计结果的计算准确性。
- **模拟提交**：多个账号同时针对同一问卷提交答卷（涵盖匿名与实名模式）。
- **统计触发**：调用 GET /api/v1/surveys/{id}/statistics 接口。
- **数据比对**：
    1.  单/多选题的 `distribution` 频次之和必须等于总有效样本量。
    2.  数字填空题的 `average_value` 需与数据库中原始 payload 的算术平均值一致。
    3.  数字填空题需像文本题一样返回逐份回答明细列表，且列表内容应与原始 payload 对应。
- **预期结果**：统计结果必须实时反映数据库最新状态，无缓存滞后。

#### 场景七：版本隔离与多版本共存
验证题目升级后，旧问卷不漂移，新旧版本可以同时被不同问卷使用。
- **创建 v1 并装配问卷 A**：问卷 A 保存时引用 `questionId=q_age, version=1`。
- **创建 v2 并装配问卷 B**：基于 v1 创建 v2，问卷 B 保存时引用 `questionId=q_age, version=2`。
- **查询问卷 Schema**：
  1. 问卷 A 返回的 `surveys.questions[].snapshot` 必须仍然是 v1 内容。
  2. 问卷 B 返回的 `surveys.questions[].snapshot` 必须是 v2 内容。
- **统计与填写验证**：向问卷 A、B 分别提交答卷，预期各自按照本问卷快照完成校验与统计，不发生串扰。

#### 场景八：题目使用查询
验证系统能否返回某个题被哪些问卷使用，并且结果覆盖共享用户创建的问卷。
- **所有者视角查询**：用户 A 创建题目并共享给用户 B，A 和 B 分别在自己的问卷中使用该题后，A 调用使用查询接口，预期结果同时包含 A、B 的问卷。
- **共享用户视角查询**：用户 B 调用同一题的使用查询接口，预期也能看到该题的问卷引用列表。
- **版本标记校验**：若 A 的问卷使用 v1，B 的问卷使用 v2，返回结果中必须标明每份问卷使用的具体版本号。
- **权限校验**：无访问权限的用户调用使用查询接口，预期失败。

#### 场景九：跨问卷统计
验证系统能否按 `questionId` 聚合多个问卷中的回答数据，并遵守权限与题型一致性约束。
- **选择题聚合**：同一选择题被多个问卷引用后，调用跨问卷统计接口，预期返回合并后的选项分布。
- **数字题聚合**：同一数字题被多个问卷引用后，预期返回跨问卷平均值、数值分布和明细列表。
- **文本题聚合**：同一文本题被多个问卷引用后，预期返回跨问卷汇总明细。
- **共享权限校验**：题目所有者和被共享用户都可以查看跨问卷统计，无权限用户调用应失败。
- **混合题型拒绝**：若同一 `questionId` 在不同问卷快照中出现多种题型，接口必须拒绝统计并返回明确错误。

#### 场景十：边界极值与压力风格测试
验证系统在临界输入与较高提交量下仍保持业务正确性。
- **边界成功用例**：
    1.  文本题输入长度恰好等于 `minLength` 与 `maxLength` 时，预期提交成功。
    2.  数字题输入数值恰好等于 `minValue` 与 `maxValue` 时，预期提交成功。
    3.  多选题选择数量恰好等于 `minSelect` 与 `maxSelect` 时，预期提交成功。
- **压力风格用例**：
    1.  使用同一已发布问卷并发提交多份答卷，预期所有请求返回成功。
    2.  并发提交完成后再次拉取统计结果，预期总答卷数、分布频次、平均值与明细列表仍与原始提交数据严格一致。
- **预期结果**：系统在边界值与较高提交量下不出现校验漂移、数据丢失或统计错误。
