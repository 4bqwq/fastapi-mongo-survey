### 问卷系统前端功能规范 (doc/07-frontend-spec.md)

本规范定义了系统前端的交互逻辑与界面要求。虽然要求界面简洁，但必须确保用户能通过图形界面完成所有业务操作，无需手动调用 API。

#### 一、 技术栈选型
*   **渲染引擎**：FastAPI + Jinja2 Templates (服务器端渲染基础结构)。
*   **样式框架**：Tailwind CSS (通过 CDN 引入，实现响应式布局)。
*   **交互逻辑**：原生 JavaScript (Fetch API) + 少量轻量级库（如用于图表的 Chart.js）。
*   **部署方式**：由 FastAPI 的 `StaticFiles` 挂载，访问根路径即可进入系统。

#### 二、 核心页面结构
1.  **身份认证页 (`/login`, `/register`)**：
    *   提供表单输入用户名与密码。
    *   登录成功后将 JWT Token 存储在 `localStorage` 或 `Cookie` 中。
    *   后续所有 API 请求需在 Header 中自动携带该 Token。

2.  **问卷看板 (`/dashboard`)**：
    *   **展示**：以列表/卡片形式展示当前用户创建的所有问卷（标题、状态、截止时间、回收量）。
    *   **操作**：
        *   “创建新问卷”按钮（弹出模态框输入标题、说明、是否允许匿名及截止时间）。
        *   “编辑”按钮（进入编辑器）。
        *   “状态切换”开关（发布/关闭问卷）。
        *   “查看统计”按钮（进入统计页）。
        *   “复制链接”按钮（获取该问卷的 C 端填写地址）。

3.  **问卷编辑器 (`/editor/{survey_id}`)**：
    *   **基础配置**：支持修改问卷标题、说明、是否允许匿名、截止时间。
    *   **题目管理**：支持从题库选择题目版本加入问卷，并在问卷内调整顺序。
    *   **题库交互**：至少支持创建题目首版本、基于旧版本创建新版本、查看某个题目的版本列表。
    *   **多态配置**：根据所选题型（单选、多选、填空、数字）动态切换配置表单。
        *   单/多选：增删选项内容。
        *   校验规则：设置必答、最少/最多选择数、字数限制、数值区间、是否强制整数。
    *   **逻辑跳转设置**：在题目下方提供“跳转规则”配置区，可选择“当本题回答为 X 时，跳转至题目 Y”。
        *   对于单选题与多选题，`X` 必须按选项行号填写，行号从 1 开始，多个值用空格分隔，例如 `1 3`。
        *   选择题跳转判断必须基于完整组合匹配，不能因部分选项重合而触发。
    *   **实时保存**：点击“保存”按钮时，只提交 `questionId + version + orderIndex` 与逻辑规则；后端负责固化快照。
    *   **显示约束**：编辑器中所有题目引用必须使用“第N题”顺序号，不显示内部题目 ID。

4.  **问卷填写页 (`/survey/{survey_id}`)**：
    *   **身份感知**：若问卷非匿名，引导用户先登录。
    *   **匿名选择**：若问卷允许匿名，提交区需提供“匿名提交”勾选项，由填写者自行决定；若未开启，则默认实名提交且不展示该开关。
    *   **动态流转**：
        *   一次仅显示一道题（或按顺序排列，但自动定位下一题）。
        *   **跳转逻辑渲染**：用户回答后，JS 立即根据后端下发的问卷快照与 `logicRules` 计算下一题索引，隐藏中间不相关的题目。
    *   **即时校验**：用户点击“下一题”或“提交”时，前端先进行格式校验（如必填检查），减少无效后端压力。
    *   **提示约束**：校验失败时，错误提示统一显示“第N题”。
    *   **数字题输入约束**：若题目未开启“仅允许整数”，前端输入控件必须允许填写小数；仅在开启该选项时才按整数输入模式限制。

5.  **数据统计页 (`/stats/{survey_id}`)**：
    *   **宏观数据**：展示总回收答卷数。
    *   **微观数据**：
        *   选择题：使用 Chart.js 展示各选项分布的饼图/柱状图。
        *   数字题：展示计算后的平均值，并同时列出每一份具体回答内容。
        *   文本题：列出所有填写的明细列表。

#### 三、 部署与启动逻辑
*   后端 `main.py` 需挂载 `app/static` 目录。
*   通过 `app/templates` 存放 HTML 文件。
*   用户访问 `http://localhost:8000` 时，后端根据 Session/Token 状态重定向至 `/login` 或 `/dashboard`。
