### 问卷系统 AI 编程通用规范 (AGENTS.md)

本文件旨在约束与指导后续 AI Agent 的编码行为，确保系统演进逻辑的一致性与可追溯性。所有参与本项目的 AI 必须严格遵守以下准则： 

#### 核心行为约束
1.  **文档驱动开发（Mandatory Docs-First）**：在编写或修改任何代码前，必须深度阅读 `doc/` 下的 03-api-spec.md、04-db-design.md 等规范文件。**严禁实现与文档描述不符的代码逻辑。**
2.  **文档与代码强同步**：在开发过程中若发现原有设计（如 API 定义或数据库模型）不合理或存在遗漏，必须遵循“先更新文档，后修改代码”的原则。必须确保 `doc/` 下的所有规格说明始终反映系统的真实实现状态。
3.  **决策透明化 (ADR)**：涉及关键架构决策或跨模块耦合调整，必须在 `doc/ADR/` 目录下创建独立的决策记录文档。
4.  **强制环境约束**：本项目唯一指定的 Python 环境与依赖管理工具为 **uv**。所有操作（如添加依赖、同步环境、运行脚本等）都必须通过 `uv` 指令完成（例如使用 `uv add` 添加包，使用 `uv run` 执行命令）。**严禁直接调用全局 pip 或系统环境来安装包或运行代码。**

#### 交付标准与风格
1.  **文档质量控制**：优先使用叙述性文本，仅在必要时使用 Markdown 格式化排版。
2.  **单元测试闭环**：功能交付必须对应 doc/05-test-cases.md 中的业务场景提供测试脚本。测试必须使用 pytest 编写，并通过 uv run pytest 执行。
 
请先阅读 `doc/` 目录下的所有文件，这些文件包含了整个项目的说明以及开发约定。
目录约定：代码主要存放于 app/，文档位于 doc/，测试代码位于 tests/。请严格遵循此结构。
